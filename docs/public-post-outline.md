# Public Post Outline

Working title:

> AI Research Skills: Stop Asking AI To Reread Your Project

## Hook

Most researchers use AI like a smart chat box. The problem is not model quality;
the problem is missing workflow memory. Every session starts by re-explaining:

- where the papers are,
- which notes matter,
- what the manuscript claims,
- what the figures show,
- what the reviewer asked for.

Skills fix that by giving the assistant durable procedures and compact context
files.

## Main Point

You do not need a perfect all-in-one research app. You can connect the tools many
researchers already use:

- Zotero for references,
- Obsidian for notes and local project memory,
- NotebookLM for source-grounded briefs,
- Claude/Codex/Gemini for AI assistance.

## The Skill Set

1. `research-hub`: make Zotero, Obsidian, and NotebookLM AI-operable.
2. `academic-writing-skills`: turn manuscript revision into claim-evidence,
   mechanism, and reviewer-response workflows.
3. `zotero-skills`: maintain Zotero libraries at scale.
4. `codex-delegate` and `gemini-delegate-skill`: use multiple AI systems without
   messy handoffs.

## Examples

### Zotero + Obsidian

Find papers, ingest metadata, create Markdown paper notes, emit a literature
matrix, and open an Obsidian dashboard.

### Obsidian + NotebookLM

Import local PDFs and notes, build a source bundle, generate a NotebookLM brief,
then verify that brief against the source bundle.

### Manuscript revision

Build `.paper/claims.yml` and `.paper/figures.yml`, then use
`academic-writing-skills` to check Abstract, Results, Discussion, and reviewer
response.

## Suggested Ending

The goal is not to replace the researcher. The goal is to stop wasting time
reloading context and start using AI on the parts of research that need judgment:
triage, synthesis, critique, revision, and reproducibility.
